1.Hugo环境搭建

建议选择Hugo.Extended

1.1前置准备

参见Installation on Windows, 需要Git，Go，Dart Sass环境

1.2常用命令

使用hugo version命令查看是否安装成功

1.2.1新建一个网站 

1

hugo new your-site-name

1.2.2新建一个post 

1

hugo new content posts/my-first-post.md

这将会根据/arcthetypes/下的模板(默认为default.md)生成md文件

1.2.3本地调试 

1

hugo server -D

-D会将草稿也渲染

1.2.4发布 

1

hugo

在site根目录下生成public文件夹

2.主题

本文使用的是FixIt

2.1FixIt安装

将主题文件夹放于/themes/下即可,一般方法如下

  • git clone
  • git submodule
  • hugo module

2.2配置

应用主题需要设置hugo.toml的theme字段即可

一般主题都会有一个模板配置文件(夹),参考Hugo Configuration和配置FixIt

3.GitHub Pages部署

参见Pages

3.1Pages仓库

主要是仓库名应为owner-name.github.io

将public下的内容上传到该仓库,即可通过https://owner's-name.github.io访问

4.GitHub Actions自动化部署

由于Pages库的可见性与网站可访问性一致.

所以在不希望公开源码的情况下,源码库和发布库可以分离,通过Github Actions实现.

最终的效果是本地源码push上源码库, 就会自动build并发布到Pages

4.1工作流

在网站目录下创建.github/workflows/hugo.yaml.

修改env中的个人信息, 再注意一下HUGO_VERSION.

name: Deploy Hugo site to Pages

on:
  push:
    branches:
      - main
  workflow_dispatch:

permissions:
  contents: write

env:
  # 推送目标仓库 格式 用户名/仓库名
  TARGET_REPOSITORY_NAME: fengjin2333/fengjin2333.github.io
  # 同步临时目录
  CLONE_DIR: tmp_public
  # 构建临时目录
  BUILD_DIR: public
  # 配置 Git 用户名
  GIT_USERNAME: fengjin2333
  # 配置 Git 邮箱
  GIT_EMAIL: feng_jin233@outlook.com

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    env:
      HUGO_VERSION: 0.141.0
    steps:
      # 步骤 1: 安装 Hugo CLI
      - name: Install Hugo CLI
        run: |
          wget -O ${{ runner.temp }}/hugo.deb https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb \
          && sudo dpkg -i ${{ runner.temp }}/hugo.deb

      # 步骤 2: 安装 Dart Sass
      - name: Install Dart Sass
        run: sudo snap install dart-sass

      # 步骤 3: 检出源码仓库
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: recursive
          fetch-depth: 0

      # 步骤 4: 使用 Hugo 构建站点
      - name: Build with Hugo
        env:
          HUGO_CACHEDIR: ${{ runner.temp }}/hugo_cache
          HUGO_ENVIRONMENT: production
          TZ: America/Los_Angeles
        run: |
          hugo \
            --gc \
            --minify \
            --destination ${{ env.BUILD_DIR }}

      # 步骤 5: 配置 Git 用户信息
      - name: Git Config
        run: |
          git config --global user.email "${{ env.GIT_EMAIL }}"
          git config --global user.name "${{ env.GIT_USERNAME }}"

      # 步骤 6: 克隆 Pages 仓库
      - name: Clone Pages repository
        run: |
          git clone --depth 1 https://${{ secrets.DEPLOYHUGO }}@github.com/${{ env.TARGET_REPOSITORY_NAME }}.git ${{ env.CLONE_DIR }}

      # 步骤 7: 清空目标目录并复制构建结果
      - name: Copy build output to Pages repository
        run: |
          rm -rf ${{ env.CLONE_DIR }}/*
          cp -rf ${{ env.BUILD_DIR }}/* ${{ env.CLONE_DIR }}/
          cd ${{ env.CLONE_DIR }}
          echo "${{ github.event.head_commit.message }} `date +%FT%T%z`" > _pub_time.html

      # 步骤 8: 提交并推送更改到 Pages 仓库
      - name: Commit and push changes
        run: |
          cd ${{ env.CLONE_DIR }}
          git add .
          git commit --message "Update from source repository: ${{ github.event.head_commit.message }}"
          git push -f -q https://${{ secrets.DEPLOYHUGO }}@github.com/${{ env.TARGET_REPOSITORY_NAME }}.git main

4.2Token

由于我们需要操作另一个repo, 因此需要在源码仓库配置Token, 参见自动部署.

注意替换

https://${{ secrets.DEPLOYHUGO }}@github.com/${{ env.TARGET_REPOSITORY_NAME }}.git

中的DEPLOYHUGO为Token名称.

5.自定义网站

5.1文章模板

Hugo中有Front Matter这个概念,用于设置文章的创建(编辑)日期,标签,标题,合集等,和配置文件一样,可以使用toml,yaml,json格式,一般放在.md文件开头.

在Front Matter里还可以使用Shortcodes,功能类似函数,可用于获取当前时间,文件名,网站名等.

为避免每次都手动编写,在/archtypes/下创建模板文件可以在hugo new时自动生成,根据文章类型有一套查找模板的逻辑,参见Archtypes.

5.2编辑网站菜单栏

hugo支持三种修改菜单的方式,详情参考Menus.

  • hugo.toml

  • define automatically

  • front matter

本文采用第一种,修改hugo.toml中的[menu]部分.

  [[menu.main]]
    identifier = "collections"

    identifier = "questbook"
    parent = ""

    post = ""
    name = "留言"
    pageRef = "/"
    title = ""
    weight = 2
    [menu.main.params]
      icon = "fa-solid fa-comments"      
  [[menu.main]]
    identifier = "friends"
    parent = ""
    pre = ""
    post = ""
    name = "友链"
    pageRef = "/"
    title = ""

    [menu.main.params]

注意以下几个字段:

  • parent:指定父级菜单

  • pageRef:指定相对于/content/的路径, 如/tags

  • name:页面显示的名称

可以使用hugo的分类机制Taxonomies

在hugo.toml中设置的分类, 可以在文章的front matter中应用, 同时在发布时会自动生成对应的文件夹, 因此[menu]中可以设置对应的pageRef.

2
3
4

[taxonomies]
  category = "categories"
  tag = "tags"
  collection = "collections"

6.插入图片

6.1存储方式

  • /static/

  • page bundle

6.1.1/static

将图片存放在/static下, 发布时就会存放到/public下

但是这要求路径为/img.png, 无法在编辑器下查看

6.1.2page bundle

参考Image processing

建立Page bundle, 将图片作为Page Resource存放

以该文章文件夹为根目录, 通过img.png即可引用
